# Code of Conduct

FirstRung Core is for building inspectable, privacy-preserving tools around
candidate-owned evidence. Project discussions should stay respectful,
practical, and focused on improving the software and its trust boundary.

## Expected Behavior

- Be direct and constructive.
- Assume contributors may have different levels of experience.
- Keep feedback specific to code, docs, behavior, evidence, and risk.
- Respect privacy: do not ask people to disclose private code, prompts, logs,
  environment values, candidate data, or customer data in public.
- Help maintain a project culture where evidence gaps are treated as things to
  clarify or improve, not as personal labels.

## Unacceptable Behavior

- Harassment, threats, or discriminatory language.
- Sexualized, violent, or demeaning comments.
- Publishing private information without consent.
- Pressuring contributors to reveal private repositories, prompts, logs, or
  credentials.
- Repeatedly derailing issues or pull requests after maintainers have set scope.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests;
block accounts; or otherwise limit participation when behavior harms the
project or its contributors.

For sensitive reports, contact maintainers privately through GitHub where
available. Do not include private security details or personal data in public
threads.
